Display: inline, block e inline-block

1. display: inline

Elementos inline (como <span>, <a> e <strong> por padrão) ficam na mesma linha um do lado do outro, ocupando apenas o espaço necessário para o próprio conteúdo — sem "quebrar" para uma nova linha.

Limitação: elementos inline ignoram completamente as propriedades width e height definidas no CSS. Além disso, margin e padding na vertical (topo e base) não empurram o conteúdo ao redor — eles até aparecem visualmente, mas não afetam o layout dos elementos vizinhos, só o horizontal (esquerda/direita) funciona como esperado. Por isso é impossível controlar o tamanho de uma caixa inline com precisão.

2. display: block

Elementos block (como <div>, <p> e <section> por padrão) sempre começam em uma nova linha e ocupam toda a largura disponível do elemento pai por padrão, mesmo que o conteúdo seja pequeno.

Limitação: mesmo definindo width: 200px, os itens nunca ficam lado a lado — cada um sempre "quebra linha" e empilha verticalmente. Isso torna o display: block puro inadequado para criar menus horizontais, grades de produtos ou qualquer layout onde os elementos precisem ficar lado a lado sem usar float, flexbox ou grid.

3. display: inline-block

O inline-block é um "meio-termo": o elemento se comporta como inline para ficar lado a lado com outros elementos, mas aceita width, height, margin e padding completos, como um bloco normal.

Limitação 1 — o "espaço fantasma": repare no pequeno espaço em branco entre os itens acima. Ele não veio de nenhuma margem definida no CSS — é o espaço em branco (quebra de linha) que existe entre as tags no próprio HTML, que o navegador renderiza como um espaço real quando os elementos são inline-block. Isso obriga a usar truques como font-size: 0 no pai ou remover as quebras de linha entre as tags no HTML.

Limitação 2 — alinhamento vertical inconsistente: por padrão, elementos inline-block se alinham pela linha de base do texto (vertical-align: baseline), o que costuma desalinhar itens de alturas diferentes de um jeito pouco intuitivo. É preciso lembrar de ajustar vertical-align manualmente (como foi feito com top nos exemplos acima) para evitar bugs visuais.

Limitação 3 — sem controle de distribuição: não existe uma forma nativa de distribuir o espaço sobrando entre os itens, centralizar com controle fino, ou reordenar a exibição sem mexer no HTML. Esse tipo de necessidade foi justamente o que levou à criação do Flexbox e do Grid, que resolvem essas limitações de forma muito mais simples e previsível.
